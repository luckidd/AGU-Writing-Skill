# AGU-Writing-Skill

Write from the author's evidence. Preserve numbers, units, notation, physical meaning, uncertainty, and claim strength. Never invent observations, instruments, events, mechanisms, statistics, references, limitations, or repository details.

## Route the task

Use the lightest sufficient mode:

- **Quick edit** — title, sentence, or short paragraph. Return the revision and only essential notes.
- **Section draft** — a complete section. Return polished text, brief structural notes, and material evidence gaps.
- **Deep audit** — logic, terminology, causality, or claim-evidence review. Return prioritized findings and a revision when requested.
- **Submission audit** — journal fit, required elements, figures, and Open Research. Return a compact pass/warn/fail table.

Identify the target journal when supplied; otherwise use Generic AGU without guessing journal-specific preferences. Identify the paper type only as far as needed: observational/event, statistical, theoretical, numerical model, data assimilation, instrument/algorithm, or forecasting/application.

## Core workflow

1. Form the bounded argument: `In [system], [evidence] supports [claim] under [conditions], while [boundary or alternative] remains.`
2. Separate direct results from physical inference: `claim -> direct evidence -> inference -> alternatives -> boundary`.
3. Draft or revise at the requested scale. Keep each paragraph focused on one job.
4. Calibrate verbs: use `show` for direct results, `indicate/suggest` for inference, and `may/could` for extrapolation. Use `demonstrate` only when major alternatives are adequately excluded.
5. Preserve the author's terminology unless inconsistent. For Chinese-to-English work, do not make the English more certain than the source.
6. If a noncritical detail is missing, write conservatively. If it changes the scientific conclusion, retain a clear placeholder such as `[sample size needed]` or state the gap.

Do not produce a claim-evidence table, outline, compliance report, or Open Research text unless the task calls for it. Do not reuse facts, numbers, or conclusions from templates.
